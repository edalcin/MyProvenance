# How to use

Here is how to document the provenance of a biodiversity data spreadsheet, from the raw data to the analysis result.

1. Create a Provenance Record with a title — for example, "Myrcia occurrences in the Atlantic Forest".
2. Register the Agents involved: you (Person), your institution (Institution) and the software used (Software), such as R or OpenRefine.
3. Record the first Activity of type Creation, which starts the lineage. E.g., the raw spreadsheet collected in the field, "raw_occurrences.csv", generated by you.
4. Add a Transformation: use "raw_occurrences.csv" and generate "darwincore_occurrences.csv", describing the cleaning and standardization of columns to the Darwin Core standard (for example, mapping "species" to scientificName and "lat/long" to decimalLatitude/decimalLongitude).
5. Record an Analysis: use "darwincore_occurrences.csv" to generate a result, such as "richness_map.png", noting the Software and parameters used.
6. Needed to fix something later? Record a new Transformation using "darwincore_occurrences.csv" and, on the generated Entity, choose its relationship to the source: "Revision" when the result is a new version of the same Entity (e.g., "darwincore_occurrences_v2.csv" fixing swapped coordinates) — pick which input Entity is being revised; or "Derivation" when the new Entity comes from the set of inputs used in the Activity, without being a direct correction of a single source (e.g., a subset filtered by taxonomic family). In the diagram, revision appears as a dashed arrow and derivation as an arrow labeled "(derivation)".
7. Watch the provenance diagram being drawn automatically and, at the end, export the .md report or the full JSON to archive or share.

> Tip: use the TD/LR button above the diagram to view it vertically (top-down) or horizontally (left-right).

## "Finalize" and "Export JSON" are different things

- **Export JSON** downloads a full copy of the record at any time, in any status (Draft or Finalized) — it works as a backup or a temporary file. It never changes the record. Without an Account, it's your only way to continue your work later: since nothing is saved on the server, export the JSON before you leave and upload it again on your next visit to pick up where you left off.
- **Finalize** is a deliberate, irreversible decision: it locks the Entities and Activities already recorded — none of them can be edited or deleted anymore, but you can still add new ones. Use it once the record is ready to be treated as an official, auditable document. Title and description remain editable even after finalization.
