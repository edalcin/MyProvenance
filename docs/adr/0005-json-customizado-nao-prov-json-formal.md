# JSON exportado é customizado, não PROV-JSON/PROV-O formal

`definicoes.md` enquadra o domínio no padrão W3C PROV, o que poderia sugerir exportar no formato formal PROV-JSON/PROV-O (namespaces, tipos qualificados). Decidimos usar um JSON customizado, com nomes de campo alinhados ao vocabulário PROV (Entidade/Atividade/Agente/Registro) mas sem compromisso de compatibilidade com a serialização formal, e com um campo `schemaVersion` para evoluir o formato sem quebrar uploads antigos. Motivo: o único consumidor do JSON é o próprio MyProvenance (download → upload de volta) — não há requisito de interoperar com outra ferramenta PROV, e a serialização formal adicionaria complexidade sem consumidor que a justifique. Alternativa rejeitada: PROV-JSON/PROV-O formal — rejeitada por over-engineering frente ao requisito real; revisar se um requisito de interoperabilidade externa surgir.
