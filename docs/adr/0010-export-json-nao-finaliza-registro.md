# Exportação de JSON não finaliza o Registro

ADR-0003 decidiu que "Finalizado" é atingido na primeira exportação do JSON ou por ação explícita do usuário. Na prática isso confundia duas funções distintas: exportar é só gerar uma cópia portátil dos dados (backup, ou o único mecanismo de continuidade de quem usa sem Conta, ADR-0009 — o navegador não persiste nada no servidor, então exportar/importar o JSON é como a sessão anônima sobrevive entre visitas); finalizar é a decisão deliberada de tornar o histórico imutável (ADR-0003). Fazer a exportação finalizar de tapa forçava o usuário sem Conta a "travar" o Registro só para tirar um backup de segurança ou levar o arquivo para outra máquina, mesmo que ainda quisesse continuar editando Entidades e Atividades depois.

Decidimos que `GET /registros/:id/export.json` (e o equivalente da sessão anônima, `sessaoAnonima.obterRegistroDetalhado` + `exportarComoArquivo`) nunca muda `status`/`finalizadoEm` — exportar é sempre uma operação de leitura, sem efeito colateral no Registro. Finalizar continua sendo exclusivamente a ação explícita do botão "Finalizar" (`POST /registros/:id/finalizar`, ou `sessaoAnonima.finalizarRegistro`). O diálogo de confirmação que avisava "exportar finaliza" (`confirm.export_finalizes`) foi removido — não há mais efeito a confirmar.

Alternativa rejeitada: manter o gatilho automático mas avisar melhor no diálogo — rejeitada porque o problema é a acoplagem em si (duas ações do usuário — "quero uma cópia" e "quero travar o histórico" — não deveriam compartilhar um único gesto), não a clareza do aviso.
