# Modo anônimo por padrão; conta opcional (username + PIN) substitui instância single-user

O modelo original (ADR-0001, ADR-0002) presumia uma instância self-hosted de usuário único, sem autenticação, onde o SQLite do servidor era a única fonte de verdade desde a primeira ação. O usuário pediu para inverter esse padrão: por privacidade e simplicidade de entrada, nada deve ser salvo no servidor até que o usuário decida explicitamente ter uma conta. Decidimos que o modo padrão (sem login) roda 100% no navegador — Registros, Entidades, Atividades e Agentes vivem só em memória na aba, nenhuma chamada de rede para as rotas de dados, e a única forma de persistir entre sessões é o export/import manual de JSON (Blob/FileReader), reaproveitando os mesmos módulos puros de validação, cardinalidade e sanitização que o servidor usa (movidos para `src/lib/` fora de `src/lib/server/` para poderem rodar no cliente). Quem quiser continuidade automática cria uma conta leve — username + PIN numérico de 6 dígitos, hash `scrypt` nativo do Node, sessão via cookie httpOnly, rate limit de 5 tentativas por 15 min — e a partir daí os Registros e Agentes daquela conta passam a persistir no SQLite como antes, agora isolados por `usuario_id`. Agente deixa de ser cadastro global (ADR original implícito) e passa a pertencer à conta, junto com Registro. O botão de criar conta só aparece depois que já existe pelo menos um Registro ou Agente na sessão anônima; ao criar conta, tudo que estava na sessão local é migrado (upsert por id, reaproveitando a rota de import já existente pelo ADR-0004, mais uma migração à parte para Agentes que ainda não foram usados em nenhuma Atividade). Alternativa rejeitada: manter SQLite como fonte de verdade sempre, com uma tela em branco só de UI — rejeitada por não atender ao pedido real (nenhum dado deve trafegar/persistir no servidor sem conta). Alternativa rejeitada: modo anônimo com validação stateless no servidor (POST sem persistir) — rejeitada em favor de rodar toda a lógica de negócio no cliente, mais simples e sem tráfego de rede nenhum enquanto anônimo. Recuperação de PIN esquecido e sincronização multi-dispositivo em tempo real ficam fora de escopo — instância self-hosted sem e-mail, perdeu o PIN cria outra conta.
