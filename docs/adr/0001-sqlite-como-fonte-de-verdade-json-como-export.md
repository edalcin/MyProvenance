# SQLite como fonte de verdade; JSON é formato de exportação/importação

`Desenvolvimento.md` exige SQLite, mas a descrição inicial do produto falava só em "salvar em JSON e baixar" / "retomar via upload do JSON", sugerindo uma ferramenta sem backend persistente. Decidimos que o SQLite no servidor é a fonte de verdade dos registros de proveniência (permitindo múltiplos registros, histórico, futura autenticação), e o JSON é apenas um formato de exportação/importação — um snapshot portátil que permite trabalhar offline, compartilhar um registro, ou retomá-lo mesmo que o servidor não tenha nada salvo. Alternativa rejeitada: app puramente client-side sem persistência no servidor, com o JSON como única fonte de verdade — rejeitada por não atender ao requisito de usar SQLite do `Desenvolvimento.md` e por perder a capacidade de listar/gerenciar múltiplos registros no servidor.
