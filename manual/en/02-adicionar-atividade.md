# Add an Activity

An Activity is what turns an empty record into a real lineage: it connects Agents, input Entities and output Entities.

## What it is and what it's for

Every action that produces or transforms data — collecting, cleaning, analyzing — is recorded as an Activity of one of three types: `Creation`, `Transformation` or `Analysis`. Each Activity is carried out by an `Agent` and can declare the Entities it uses and the ones it generates.

## When to use it

Any time a new piece of data enters the lineage (Creation) or existing data gets processed (Transformation/Analysis).

## Step by step

1. On the record, click `Add Activity`.
2. Choose the type from the tabs: `Creation`, `Transformation` or `Analysis`.
3. Select the responsible `Agent` — the field has autocomplete search; if the Agent doesn't exist yet, type the name and click `Create "<name>"` to register it inline without leaving the form.
4. Fill in `Date` and `Description`.
5. Type-specific fields:
   - **Creation**: `Location` and `Tool or Software` (where the data came from and with what).
   - **Transformation/Analysis**: check the `Entities used`; fill in `Process` (what was done), parameters (`Key`/`Value`) and `Execution environment — operating system`.
6. Under `Generated entities`, click `Add generated entity` for each output, filling in `Name`, `Format`, `License`, `Location` and `Description`.
7. For each entity generated by a Transformation/Analysis, choose the `Relationship to source`: `None`, `Derivation` or `Revision` (the latter asks for `Revision of`, pointing to which used entity it replaces).
8. Click `Add Activity` to save.

## Cardinality by type

- **Creation** — generates 1 or more Entities; uses no existing Entity.
- **Transformation** — uses 1 or more Entities; generates 1 or more Entities.
- **Analysis** — uses 1 or more Entities; generates 0 or more Entities (an Analysis can exist purely to verify data, with no new output).

> 🔗 **Pattern:** the Activity is a `prov:Activity`; using an existing Entity is `prov:used`; generating a new one is `prov:wasGeneratedBy`; the responsible Agent is `prov:wasAssociatedWith`. `Derivation` maps to `prov:wasDerivedFrom`, `Revision` to `prov:wasRevisionOf`.

> 💡 **Tip:** more than one Entity can be generated by the same Activity — useful when a single process produces several output files (e.g., a table and a chart).

> ⚠️ **Warning:** once a record is finalized, existing Activities can no longer be edited or deleted — only new Activities can be added. See [Finalize](/como-usar/04-finalizar).
